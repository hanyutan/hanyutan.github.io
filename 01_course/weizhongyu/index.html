  • Search Problems
    • state space 状态空间
    • successor function (with actions, costs)
    • start state 起始空间
    • goal test 检测是否达到目标状态
  • Uninformed Search 无信息搜索算法
    • Depth-First Search 深度优先
    • Breadth-First Search 广度优先

    • Pac-man吃豆子

同一个场景，目标不同，问题复杂度也不一样

State Space Graphs状态空间图 - A mathematical representation of a search problem - Nodes are (abstracted) world configurations - Arcs represent successors (action results) - The goal test is a set of goal nodes (maybe only one) - 状态不会重复

Search Trees搜索树 - The start state is the root node - Children correspond to successors - Nodes show states, but correspond to PLANS that achieve those states - 状态会重复

Depth-First Search 深度优先

Strategy: expand a deepest node first 深度最大的先弹出 Implementation: Fringe is a LIFO stack 弹出的时候/扩展的时候做目标检测goal test

深度 Frontier观察到的所有节点 Pop弹出 Expan扩展节点
0 S S d e p
1 1 1 d e p d b c e
1 1 2 2 2 e p b c e b a

算法评价（几个指标） - 完备性Complete：有答案就一定能找到 - 最优性Optimal：是否能找到最优方法 - 时间复杂度Time complexity：找到答案最差情况所经历的节点 - 空间复杂度Space complexity：frontier的结构

Breadth-First Search 广度优先

Strategy: expand a shallowest node first Implementation: Fringe is a FIFO queue

比较 广度优先：存储空间更大，完备，最优，稳定

Iterative Deepening 二者结合 每次限定搜索深度为d，进行深度优先搜索，然后迭代增加深度

Uniform-Cost Search 代价一致 Strategy: expand a cheapest node first 弹出代价最小的节点 Fringe is a priority queue (priority: cumulative cost 根节点到本节点的路径消耗)

Uniform Cost Search (UCS) Properties 定义虚拟树高，剩余与广度优先思想一致
