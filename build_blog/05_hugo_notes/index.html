Hugo Notes

Hugo官方教程：https://gohugo.io/getting-started/

有人自己创建的Hugo中文文档：https://www.gohugo.org/

Hugo主题模板：https://themes.gohugo.io/

Create New Site

hugo new site Hugo_Blog     # 新建网站
cd Hugo_Blog                # 切换目录
git init                    # 初始化respository

详细指南请参考：https://gohugo.io/getting-started/quick-start/

假定设定目标文件夹D:/Github用于存储网页，右键打开Git Bash。使用hugo命令new site来新建网站Hugo_Blog（命名自定义）。
然后hugo会在D:/Github/Hugo_Blog自动生成这样一个目录结构（大致样子）：

文件(夹)名 用途
archetypes 为不同类型定义默认的头文件FrontMatter
content 存放写好的文章，markdown文件（.md文件）
data 数据文件，一般用不上
layouts 存放网站的模板文件（toml文件，或json、yaml格式）
static 存放image/css/js等静态资源
themes 存放下载的Hugo主题
config.toml 网站的配置文件

Install Themes

# 下载安装Ananke主题到D:/Github/Hugo_Blog/themes/ananke
git submodule add https://github.com/bep/docuapi.git themes/docuapi
# 把主题的demo样例文件复制过来
cp -r themes/ananke/exampleSite/. ./

网址可以从选中主题的Home Page（即github仓库+.git获得）代码后面部分表明安装到哪个文件夹。

themes/ananke/exampleSite是一个示例站点，我们可以直接拿来使用，所以需要copy里面的相关配置文件到我们自己的站点里。 右键用文本编辑器Notepad++打开D:/Github/Hugo_Blog/config.toml文件， 自定义title（会显示在网页标签上），修改baseURL为自己的github.io地址，修改主题theme为ananke，删掉themesDir = "../.."。 然后记得打开文件D:/Github/Hugo_Blog/themes/ananke/theme.toml，修改第11行min_version = "0.30.2"，需要加上双引号，否则系统把版本号当作float类型会报错。 自定义配置：可以自己研究config.toml文件以及themes/ananke/layouts文件夹力度内容，进行相应修改。

Create New Post

# 创建markdown文件
hugo new post/my-first-post.md

hugo new指令创建markdown文件会自动添加content/路径前缀， 打开D:/Github/Hugo_Blog/content/post/my-first-post.md进行编辑。 使用---或+++包围起来的部分是各个静态网页编辑器SSG统一的YAML开头FrontMatter，可以自行修改指明文章标题、创建时间。 注意发布时应将draft改为false，否则为草稿状态。

Build Static Site

hugo server --watch        # 创建本地网站
hugo                       # 建立静态网页

可以使用hugo server命令创建本地网站，–watch或者-w选项打开的话，将会监控到Markdown文件/主题配置文件的改动从而自动去刷新浏览器， 不需要自己手工去刷新浏览器，非常方便。浏览器里打开：http://localhost:1313查看本地网站。

使用hugo建立静态网页，所有静态页面会生成到public目录，将public文件夹里的全部文件push到对应的分支上。 （例如<username>.github.io的master分支上，或者其他项目repository的gh-pages上） 之后在浏览器中输入https://<username>.github.io查看个人主页，或者https://<username>.github.io/repository_name查看项目主页。
