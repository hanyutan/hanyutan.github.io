Introduction

如果你不会Web前端开发，也不愿意花钱买服务器，懒洋洋地做个伸手党，只希望专注于内容创作…… 那么Github Pages + Hugo绝对适合你！

Why Github Pages?

免费！创建和管理都很方便！

只需在Github上创建一个repository，然后将Hugo生成的静态页面相关文件push到项目上。 （个人主页<username>.github.io的话push到默认的master分支上，普通repository则push到该项目的gh-pages分支上并保证根目录有一个index.html文件）

Push的方法有多种，说一些我接触到的：

  • 使用Git命令（最原始，最基本，灵活度高）
  • 下载GitHub桌面版，2次按键搞定（最简单，灵活度低）
  • 基于Git使用sublime上传
  • Eclipse里自带Git，但需要Java环境，jre或jdk都行。

Why Hugo?

目前流行的几个静态网站生成器（SSG——Static Site Generator）：Hugo/Hexo/Jekyll。

Hugo官方主页：https://gohugo.io

  • Hugo是使用Go语言编写的静态网站生成器。
  • 相对于Jekyll复杂的安装设置来说，Hugo进行需要下载一个二进制文件（hogo.exe）即可轻松调试和生成静态页面。
  • Hugo生成静态页码的效率很高，速度上来说Hugo>Hexo>Jekyll。
  • Hugo自带watch的调试模式，可以在修改Markdown的文章内容的同时，自动更新浏览器页面呈现最终效果。
  • 缺点是主题模板相对其他两个来说数量比较少。
  • 用的人感觉还不算特别多，中文社区支持也不是很多。

Preparation

需要拥有自己的GitHub账号、创建个人主页用的仓库、安装Git和Hugo、一定的Markdown和HTML基础。 GitHub Desktop和Go语言可以根据自身需要选择性地安装。 另外最好还要有一款支持语法高亮的文本编辑器/代码编辑器，例如Sublime Text、Notepad++等等。

GitHub Account & Repository

进入GitHub官方主页https://github.com/注册自己的GitHub账户，登陆个人主页http://github.com/username， 右上角点击+号新建仓库（New Repository），在Repository name一栏中为仓库命名为<username>.github.io。 注意username需要与注册时保持一致。Initialize this repository with a README先不用勾选。

Install Git & Hugo

Install Git

Git下载安装网站：https://git-scm.com/download

安装完成后，在开始菜单里找到Git -> Git Bash，蹦出一个命令行窗口说明Git安装成功。

Git Bash命令行输入

git config --global user.name "Your Name"
git config --global user.email email@example.com

因为Git是分布式版本控制系统，所以，每个机器都必须自报家门：你的名字和Email地址。 注意git config命令的–global参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置

如果想要修改用户主目录的默认路径，可以修改环境变量：在用户变量中新建，名称为HOME，地址自定义。 其实不修改默认路径也ok，每次直接在你的目标文件夹里右键Git Bash Here，即可在当前目录中创建命令行窗口。

Install Hugo

详细安装指南请参考：https://gohugo.io/getting-started/installing/

我用的Windows系统，两种常用方法（推荐第二种）：

  1. 在Hugo Releases上下载最新版的二进制程序：https://github.com/spf13/hugo/releases，下载后解压，然后把解压目录添加到环境变量path里。
  2. 使用Scoop包管理器来管理Windows软件，一行代码搞定安装和卸载，干净又迅速。操作如右侧代码栏：

Windows PowerShell命令行输入

# Windows10在开始菜单键右击选择Windows PowerShell，依次输入下列命令
set-executionpolicy remotesigned -scope currentuser	
$env:SCOOP='D:\Scoop'      # 这里可以替换到你自定义的安装目录
[environment]::setEnvironmentVariable('SCOOP',$env:SCOOP,'User')
iex (new-object net.webclient).downloadstring('https://get.scoop.sh')

# 一行代码即可安装和卸载
scoop install hugo         # 安装Hugo，D:\Scoop\apps出现hugo文件夹
scoop uninstall hugo       # 卸载Hugo超简单
scoop uninstall scoop      # 卸载Scoop超简单

Git Bash命令行输入

# 验证是否安装成功：
# Win+R输入cmd回车，打开命令行窗口/或者最好直接打开Git Bash
hugo help                  # 如果正常运行则安装成功

Quick Start

不想看后面的Git/Hugo/Markdown Notes的话，可以直接按照Quick Start的操作来迅速建立一个个人Blog。

打开Github Desktop，登陆账号后Clone a repository from the Internet，选择你刚刚创建的仓库<username>.github.io，下面确定clone到本地的地址，例如D:/Github/hanyutan.github.io。

在D:/Github目录下右击Git Bash Here，如右侧代码栏输入命令。

hugo new site Hugo_Blog    # 新建网站
cd Hugo_Blog               # 切换目录
git init                   # 初始化仓库

# 下载安装Ananke主题，并把demo相关文件复制过来
git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke
cp -r themes/ananke/exampleSite/. ./

右键用文本编辑器Notepad++打开D:/Github/Hugo_Blog/config.toml文件， 自定义title（会显示在网页标签上），修改baseURL为自己的github.io地址，修改主题theme为”ananke”，删掉themesDir = "../.."。 然后记得打开文件D:/Github/Hugo_Blog/themes/ananke/theme.toml，修改第11行min_version = "0.30.2"，加上双引号，否则系统把版本号当作float类型会报错。 自定义配置：可以自己研究config.toml文件以及themes/ananke/layouts文件夹力度内容，进行相应修改。

# 创建本地网站，可在http://localhost:1313/查看效果
hugo server --watch
# 创建markdown文章，打开D:/Github/Hugo_Blog/content/post/my-first-post.md进行编辑
hugo new post/my-first-post.md
# 建立静态网页
hugo

成功建立静态网页后，将public目录里的全部内容拷贝到刚才通过GitHub Desktop克隆的本地仓库D:/Github/hanyutan.github.io中。 打开GitHub Desktop，左下角Summary栏写一点此次提交的备注，例如”add ananke theme”，然后点击Commit to master。 最后点击导航栏的Publish branch提交到远程仓库。现在就可以打开你的个人主页网址<username>.github.io欣赏成果啦！

日常管理：添加/修改/删除markdown文件的同时可以通过hugo server --watch随时在http://localhost:1313/上查看效果。 修改完成后，hugo一下，复制public文件夹内容到本地仓库，提交修改。
