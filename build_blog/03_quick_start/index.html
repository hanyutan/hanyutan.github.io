Quick Start

不想看后面的Git/Hugo/Markdown Notes的话，可以直接按照Quick Start的操作来迅速建立一个个人Blog。

打开Github Desktop，登陆账号后Clone a repository from the Internet，选择你刚刚创建的仓库<username>.github.io，下面确定clone到本地的地址，例如D:/Github/hanyutan.github.io。

在D:/Github目录下右击Git Bash Here，如右侧代码栏输入命令。

hugo new site Hugo_Blog    # 新建网站
cd Hugo_Blog               # 切换目录
git init                   # 初始化仓库

# 下载安装Ananke主题，并把demo相关文件复制过来
git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke
cp -r themes/ananke/exampleSite/. ./

右键用文本编辑器Notepad++打开D:/Github/Hugo_Blog/config.toml文件， 自定义title（会显示在网页标签上），修改baseURL为自己的github.io地址，修改主题theme为”ananke”，删掉themesDir = "../.."。 然后记得打开文件D:/Github/Hugo_Blog/themes/ananke/theme.toml，修改第11行min_version = "0.30.2"，加上双引号，否则系统把版本号当作float类型会报错。 自定义配置：可以自己研究config.toml文件以及themes/ananke/layouts文件夹力度内容，进行相应修改。

# 创建本地网站，可在http://localhost:1313/查看效果
hugo server --watch
# 创建markdown文章，打开D:/Github/Hugo_Blog/content/post/my-first-post.md进行编辑
hugo new post/my-first-post.md
# 建立静态网页
hugo

成功建立静态网页后，将public目录里的全部内容拷贝到刚才通过GitHub Desktop克隆的本地仓库D:/Github/hanyutan.github.io中。 打开GitHub Desktop，左下角Summary栏写一点此次提交的备注，例如”add ananke theme”，然后点击Commit to master。 最后点击导航栏的Publish branch提交到远程仓库。现在就可以打开你的个人主页网址<username>.github.io欣赏成果啦！

日常管理：添加/修改/删除markdown文件的同时可以通过hugo server --watch随时在http://localhost:1313/上查看效果。 修改完成后，hugo一下，复制public文件夹内容到本地仓库，提交修改。
