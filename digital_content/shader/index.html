在Unity2018版本，Unity为了提高引擎的画面表现，推出了High Definition Render Pipeline 高清晰渲染管线，简称HDRP。

[ToC]

Description

Concepts

Shader（着色器）：专门用来渲染图形的一种技术，可以自定义显卡渲染画面的算法。其实就是一段规定好输入和输出的代码，告诉GPU如何绘制模型每一个顶点的颜色以及最终每个像素点的颜色。（ 负责将输入的Mesh（网格）以指定的方式和输入的贴图或者颜色等组合作用，然后输出渲染器能够读懂的点和颜色的对应关系；绘图单元可以依据这个输出来将图像绘制到屏幕上）。

Material（材质）：输入的贴图或者颜色等，加上对应的Shader以及对Shader的特定参数设置，将这些打包存储到一起得到的就是Material。之后将其赋予合适的renderer（渲染器）进行渲染（输出）。

Type

Shader大体上分为两类：顶点 Shader 和 像素 Shader。

表面着色器（Surface Shader）简单，类比傻瓜相机

片段着色器（Fragment Shader）较难，目的是可以在比较低的层级上进行更复杂（或者针对目标设备更高效）的开发

Unity界面中在Project面板点击Create/Shader中包含：

  • Standard Surface Shader 标准表面着色器
    模拟材质与灯光之间的关系
  • Unlit Shader 最基本的顶点片段着色器
    Vertex/Fragment Shader 不受光照影响
  • Image Effect Shader 也是顶点片段着色器，针对后处理而定制的模板
    后处理：泛光 调色 景深 模糊
  • Computer Shader 直接将GPU作为并行处理器加以利用
  • Shader Variant Collection 变体收集器

Shader代码框架结构 []代表可选

Shader "Path/Name"
{
    [Properties] # 属性，定义材质球面板中的贴图和参数等
    SubShader    # 可以包含多个SubShader，对应高中低不同的硬件配置，但至少有一个
        Pass     # 渲染一次模型，SubShader里可以有多个Pass
    [fallBack]   # shader在机器上不支持时，自动切换到这里备用的shader
    [CustomEditor] # 自定义界面，材质面板的显示结果
}

Properties

Properties可以理解为是材质与Shader的连接通道，我们在材质面板上需要设置的内容都必须通过Properties来实现并暴露。

属性的写法有个通用的格式：
[Attribute]_Name ("Display Name",Type) = Default Value

学习参考

Type: 属性的类型，常用的有以下几种：Color颜色 Int整数 Float浮点数 Vector四维数 2D纹理 3D纹理 Cube立方体纹理

Properties{
    // Color 颜色 属性是四维变量RGBA 红/绿/蓝/透明 in(0,1)
    _Color("我是Color",Color) = (1,1,1,1)
    //HDR
    //Int 整数
    _Int("我是Int", Int) = 1
    //Float 浮点数
    _Float("我是Float", Float) = 0.5
    // Range 限制区间，增加滑动条选择
	_Float("我是Float", Range(0,1)) = 0.5
    // PowerSlider 参数值越大 滑动条曲度变化变化越大
    [PowerSlider(3)]_Float("我是Float", Range(0,1)) = 0.5
    //IntRange 只能生成整数
    [IntRange]_Float("我是Float", Range(0,1)) = 1
    //Toggle 开关 1-0
    [Toggle]_Float("我是Float", Range(0,1)) = 1 
    //Enum 枚举 下拉列表
    [Enum(UnityEngine.Rendering.CullMode)]_Float("我是Float", Float) = 1 
    // Vector 四维变量
    _Vector("我是Vector", Vector) = (0,0,0,0)
    // 2D
    _MainTex("我是2D纹理", 2D) = "white" {}
    // NoScaleOffset 隐藏调节参数
    [NoScaleOffset]_MainTex("我是2D纹理", 2D) = "white" {}  
}

学习参考

渲染一次模型， 代码片断是由CGPROGRAM开始，由ENDCG结束

#pragma声明所需要的顶点着色器与片段着色器

顶点着色器用于处理顶点，每个顶点都会执行一次顶点着色器

片段着色器用于处理像素，每个像素都会执行一次片段着色器

SubShader

        Pass

            #pragma fragment frag    //定义好片段着色器

Shader Graph

About Shader Graph link 着色器视图

Description

Shader Graph enables you to build shaders visually and get instant feedback, instead of writing code.

Shader Graph is only compatible with the Scriptable Render Pipeline (SRP) : the Universal Render Pipeline (URP) or the High Definition Render Pipeline (HDRP). The legacy built-in render pipeline does not support Shader Graph.

URP 代替了原先的 Lightweight Render Pipeline (LWRP)

How to use

Install

Unity -> Window -> Package Manager -> Shader Graph -> Install

Creating a new Shader Graph Asset

Project window: click Create -> Shader, then select your desired type of Shader Graph.

Unity currently supports: PBR Graph, Sub Graph, Unlit Graph

HDRP-specific graph (if using HDRP): Decal Graph, Fabric Graph, Hair Graph, Lit Graph, StackLit Graph, Unlit Graph

Double-click your newly created Shader Graph Asset to open it in the Shader Graph window.

Shader Graph Window

consists of the Master Node, the Preview Window, and the Blackboard.

Master Node The final connection that determines your shader output.

Preview Window An area to preview the current shader output. Here, you can rotate the object, and zoom in and out. You can also change the basic mesh on which the shader is previewed.

Blackboard An area that contains all of the shader’s properties in a single, collected view. Use the Blackboard to add, remove, rename, and reorder properties.

https://unity.com/cn/shader-graph

Volume组件

在Hierarchy视图中，看见Volume Settings物体上添加有Volume组件, 是HDRP中对一定范围内的光照，雾，以及阴影进行管理的一个组件。 只有当摄像机进入其影响范围内，才会出现对应的效果。

子组件：

  • HD Shadow Settings（高清阴影设置）主要是用于控制Volume组件范围内的阴影质量相关。
  • Visual Environment(可视环境)，用于控制天空盒以及远处雾气的效果，不添加或者关闭将不会看见相应的视觉画面。其中有2个选项，用来设置天空的类型，以及雾的类型。 默认情况下，分别为Procedural Sky与Exponential Fog。
    • Gradient Sky（渐变天空）当Visual Environment组件Sky Type选择Gradient Sky类型选项时，添加此组件，可以进行相应的设置。 Gradient Sky可以定义三种颜色，顶部，中部和底部，HDRP会在它们之间进行插值以创建渐变天空盒。
    • Procedural Sky（程序化天空），使用步骤同上，但是设置的效果不同。 Procedural Sky可以设置太阳大小，地面颜色和天空色调来自定义天空盒。
    • HDRI Sky是一种使用立方体贴图纹理的天空盒。Unity商店提供了免费HDRP使用的立方体纹理包Unity HDRI Pack。 Asset store搜索下载导入即可使用。
    • Linear Fog（线性雾）当Visual Environment组件Fog Type选择Linear Fog类型选项时，添加此组件，可以进行相应的设置。 Linear Fog组件将会根据提供的参数，来设置场景中的雾。线性雾的密度将随视距和世界空间高度线性增加。
    • Exponential Fog（指数雾）指数雾密度将随着视距和世界空间高度呈指数增长，具体取决于提供的参数。

新版相机组件与灯光组件

HDRP的新相机组件，除了在原先旧版相机的基础上增加了新的功能，并进行了分级排版。而且还必须与HD Additional Camera Data（高清相机附加数据）组件同时使用，才能得到正确的相机效果。

新版的灯光组件改动与相机类似，只不过是需要灯光组件物体上同时挂载HD Additional Light Data（高清灯光附加数据）与Additional Shadow Data（阴影附加数据）这2个组件，才能正确使用新的灯光组件。

同时新的灯光组件，还增加了新的灯光类型-Line。其效果类似于发光灯管，值得注意得是使用此类型的灯光，将不会对受到灯光影响的物体投射阴影。

高清着色器 Lit

新版的高清着色器Lit旨在让开发者使用最少的配置，创建更加逼真的材质。而在HDRP工程中，创建新材质的时候默认也是使用的Lit着色器。

显卡： 驱动：卖硬件显卡的会自己写驱动程序 应用程序：头疼不同的驱动程序给出的接口不一样 没有游戏引擎也可以直接调用OpenGL 游戏引擎相当于再往上一层打包的工具包，其实还是调用OpenGL 游戏引擎：Unity3d（普及率高，渲染管线正在改好） 虚幻（画质很好，3d渲染管线很厉害，但难用）

光照

关于漫反射的光照模型，常见的是兰伯特(Lambert)和半兰伯特(Half-Lambert)

Lambert的光照公式是(C-light · M-diffuse)max(0,n·I)，

即光照颜色 * 漫反射颜色 * max(0,法向量*光照方向)

注意光照方向是指向光的方向，即入射光向量的逆向量

fixed3 ambient = UNITY_LIGHTMODEL_AMBIENT.xyz;
                
fixed3 worldNormal = normalize(mul(v.normal,(float3x3)unity_WorldToObject));
               
fixed3 worldLight = normalize(_WorldSpaceLightPos0.xyz);
                
fixed3 diffuse = _LightColor0.rgb*_Diffuse.rgb*saturate(dot(worldNormal, worldLight));
                
o.color = ambient + diffuse;
  • ambient：代表环境光，这个环境光是由系统自动提供的，通过UNITY_LIGHTMODEL_AMBIENT.xyz获取。最后我们计算出的漫反射光的颜色需要跟环境光相加得到最后的颜色值。
  • worldNormal：代表该顶点所在物体表面的法线，可以理解为物体表面的朝向，对应的就是漫反射公式里的表面法线（n^）
  • worldLight：代表的是光源方向，也就是公式里的光源（l^）

  • diffuse：漫反射颜色值，对应的就是漫反射公式里的C diffuse

  • normalize：有语言基础的同学应该都知道，是归一化的意思，就是把原向量转化为长度为1，并且方向不变的向量。

  • mul:支持向量于矩阵相乘，用来做坐标系变换，比如从物体坐标系转换成世界坐标系。

  • saturate:截取0-1的范围，跟之前介绍的max（0，n * l）取其中最大值的效果是一样的，是Cg提供的函数

  • dot:是点乘的意思也叫点积，两个向量的点乘=两个向量的长度*夹角的余弦值，即A*B = |A||B|cos(θ)。
